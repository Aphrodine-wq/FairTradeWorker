# Expansion Playbook

## Expansion Decision Framework

Before entering any new market, answer:

1. **Do we have a local champion?** Someone like James/Josh in that market who knows contractors personally.
2. **Is the licensing framework compatible?** Same state = easy, new state = compliance work.
3. **Can we reach 10 contractors in 30 days?** If not, market is too cold.
4. **Is the home value high enough?** Under $200K avg = more DIY, less platform value.
5. **Can Oxford support this expansion financially?** Do not expand before Oxford is profitable.

### Success Criteria Before Expanding

- 10+ active contractors using tools weekly
- 5+ jobs/week flowing through marketplace (if Route B active)
- 50+ Pro subscribers
- Positive unit economics (LTV/CAC > 3x)
- NPS score of 40+ from contractor surveys

### Why Oxford First

- James lives there and knows contractors personally
- Population 27,448 — small enough that 10-15 contractors covers the market
- Ole Miss drives steady housing demand (faculty housing, student rentals, alumni vacation homes)
- $400K+ average home values in desirable neighborhoods justify professional contractor work
- Can validate product-market fit with minimal marketing spend
- Mississippi contractor licensing is straightforward (MSBOC for >$50K residential)
- Zero direct competitors in the digital contractor platform space

### Launch Playbook for Oxford

- Personally visit 15-20 contractors James knows

**Weeks 5-8: Activation**

- Post in Oxford MS Facebook groups (Buy/Sell, Oxford Neighbors) about finding licensed contractors through FTW

**Weeks 9-12: Growth**
- Host a lunch for local contractors — demo the platform, share roadmap
- Launch homeowner-facing features (find a contractor, request estimates)

- Target: 15 Pro contractors, 100+ homeowner signups

**Weeks 13-24: Scale**
- Activate QuickBooks integration for connected contractors
- Launch marketplace features (job posting, bidding)
- Begin tracking metrics religiously (WAU, jobs/week, estimate accuracy, contractor satisfaction)
- Target: 50 Pro contractors, 500+ homeowner signups, $5K MRR

### Tennessee Licensing Compliance

Tennessee contractor licensing is managed by the Tennessee Board for Licensing Contractors (TBLC):

| Requirement | Details |
|-------------|---------|
| License threshold | Required for any project over $25,000 (lower than MS's $50,000) |
| License types | BC-A (unlimited), BC-B (up to $1.5M), BC-b (up to $500K), BC-C (specialty) |
| Financial requirements | BC-A: $100K net worth + $25K working capital. BC-b: $25K net worth + $10K working capital |
| Insurance | General liability required. Workers comp required if 5+ employees |
| Exam | Required — Tennessee contractor licensing exam |
| Reciprocity with MS | None — Mississippi license does not transfer to Tennessee |
| Continuing education | 6 hours every 2 years |
| FTW impact | Must verify Tennessee license numbers, different license tiers, adjust contractor onboarding flow |

**FTW platform changes needed for Tennessee:**
1. Add Tennessee license verification to contractor onboarding
2. Update license type dropdowns (BC-A, BC-B, BC-b, BC-C vs Mississippi's residential/commercial)
3. Add Tennessee-specific insurance requirements to verification checklist
4. Update ConstructionAI with Tennessee pricing data (labor rates, material costs differ from MS)
5. Add Tennessee to location filtering in marketplace

### Technology Considerations for Multi-Market Support

**Location filtering:**
- Add `market` field to all jobs, contractors, and homeowners
- Marketplace search defaults to user's market but allows cross-market search
- Contractor profiles show service area (which markets they cover)
- Some contractors work multiple markets (Oxford + Tupelo, Memphis + suburbs)

**Market-specific pricing data:**
- ConstructionAI needs separate pricing models per market
- Labor rates differ significantly: Nashville $45-65/hr, Jackson $30-45/hr, Oxford $35-50/hr
- Material costs vary less but delivery costs differ by market
- Strategy: train market-specific ConstructionAI models or use market as a feature in the base model

**Regional ConstructionAI models:**
- Phase 1: Single model with market as an input parameter
- Phase 2: Fine-tuned models per state (MS model, TN model, AL model)
- Phase 3: Metro-specific models for high-volume markets (Nashville, Memphis)
- Training data source: actual FTW estimates + outcomes from each market

**Multi-market analytics:**
- Admin dashboard showing per-market metrics (contractors, jobs, revenue, CAC, LTV)
- Cross-market comparison (which market is growing fastest, which has best unit economics)
- Market-level P&L (can each market sustain itself?)
